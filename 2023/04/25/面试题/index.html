js 的数据类型有哪些 -> 6 + 3 => bigInt

数据类型分为两种

1.基本数据类型
  基本数据类型包括(number,string,boolean,undefined,null,Es6新增(symbol))
2.引用数据类型
  引用数据类型包括(function,Object,Array)
BigInt 是 ES2020 中引入的一种新的数据类型，用于表示任意精度整数。它可以处理超过 Number 类型所能表示的最大整数范围，从而解决了 JavaScript 中某些数值计算的精度问题。

在使用 BigInt 时，我们需要在数字后面添加一个大写的 n 来标识它是一个 BigInt 类型的值。例如：

javascript
const bigNum = 1234567890123456789012345678901234567890n;
需要注意的是，BigInt 值不能与普通数字相加、相减等运算，必须显式地将其转换为 BigInt 类型。以下是一些示例：

javascript
// 将数字转换为 BigInt 类型
const a = BigInt(10);

// 使用 `n` 后缀声明 BigInt 类型
const b = 12345n;

// BigInt 与 BigInt 运算
const c = a + b;

// BigInt 转换为普通数字
const d = Number(c);
由于 BigInt 是一种新的数据类型，因此不是所有的浏览器和 JavaScript 引擎都支持它。建议在使用之前，先检查浏览器或引擎是否支持该特性。

总之，BigInt 是一种非常有用的数据类型，可以帮助我们处理需要高精度计算的场景，如密码学、日期时间计算等。


vue3 中 setup 在什么时候触发

在 Vue 3 中，组件的 setup 函数是在组件实例创建之前执行的。具体来说，在组件被挂载到 DOM 上之前，Vue 会先执行 setup 函数，将 setup 函数返回的数据作为响应式对象注入到组件实例中，然后再开始执行组件的生命周期钩子函数。

setup 函数的执行时机和 Vue 2 中的 beforeCreate 生命周期钩子类似，但有一些区别：

setup 函数的参数包括了组件的 props、attrs、context 等属性，并且不能访问组件实例（即 this）；
setup 函数中可以使用 Vue 3 提供的新 API，如 ref、reactive、computed 等，用于创建响应式数据；
setup 函数需要返回一个对象，该对象中的属性和方法将被注入到组件实例中，可以在组件中通过 this.xxx 的方式进行访问。
总之，在 Vue 3 中，setup 函数是一个非常重要的概念，它负责创建并初始化组件的状态，并将其注入到组件实例中。由于 setup 函数的执行时机很早，因此可以在其中进行一些预处理或者初始化操作，从而减少在组件生命周期钩子中的计算量，提高性能和效率。
