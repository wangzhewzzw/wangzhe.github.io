js 的数据类型有哪些 -> 6 + 3 => bigInt

数据类型分为两种

1.基本数据类型
  基本数据类型包括(number,string,boolean,undefined,null,Es6新增(symbol))
2.引用数据类型
  引用数据类型包括(function,Object,Array)
BigInt 是 ES2020 中引入的一种新的数据类型，用于表示任意精度整数。它可以处理超过 Number 类型所能表示的最大整数范围，从而解决了 JavaScript 中某些数值计算的精度问题。

在使用 BigInt 时，我们需要在数字后面添加一个大写的 n 来标识它是一个 BigInt 类型的值。例如：

javascript
const bigNum = 1234567890123456789012345678901234567890n;
需要注意的是，BigInt 值不能与普通数字相加、相减等运算，必须显式地将其转换为 BigInt 类型。以下是一些示例：

javascript
// 将数字转换为 BigInt 类型
const a = BigInt(10);

// 使用 `n` 后缀声明 BigInt 类型
const b = 12345n;

// BigInt 与 BigInt 运算
const c = a + b;

// BigInt 转换为普通数字
const d = Number(c);
由于 BigInt 是一种新的数据类型，因此不是所有的浏览器和 JavaScript 引擎都支持它。建议在使用之前，先检查浏览器或引擎是否支持该特性。

总之，BigInt 是一种非常有用的数据类型，可以帮助我们处理需要高精度计算的场景，如密码学、日期时间计算等。

数据拷贝-拷贝的方法有哪些

拷贝方法有两种：(浅拷贝和深拷贝)

浅拷贝：
  创建一个新对象，这个对象有着原始对象属性值的一份精确拷贝。如果拷贝的属性是基本类型，拷贝的就是基本类型的值，如果属性是引用类型，拷贝的就是内存地址，所以其中一个对象改变了这个地址，就会影响另一个对象。

深拷贝：
  会拷贝所有的属性，并拷贝属性指向的动态分配的内存。当对象和他所引用的对象一起拷贝的时候就发生深拷贝。深拷贝相比于浅拷贝速度较慢并且花销较大。拷贝前后两个对象互不影响。

递归拷贝（）

function deepClone(obj) {
  // 检查输入参数是否为对象或数组类型
  if (typeof obj !== 'object' || obj === null) {
    return obj
  }

  let result
  if (Array.isArray(obj)) {
    result = []
    for (let i = 0; i < obj.length; i++) {
      result.push(deepClone(obj[i]))
    }
  } else {
    result = {}
    for (let key in obj) {
      if (obj.hasOwnProperty(key)) {
        result[key] = deepClone(obj[key])
      }
    }
  }

  return result
}

作用域

什么是作用域：
  就是变量和函数生效的区域或者说集合

作用域分为哪些：
  块级作用域：
    ES6引⼊了 let 和 const 关键字,和 var 关键字不同，在⼤括号中使⽤ let 和 const 声明的变量存 在于块级作⽤域中。在⼤括号之外不能访问这些变量
  局部作用域：
    (又叫函数作用域)听名字就知道是在函数内部作用的区域，如果⼀个变量是在函数内部声明的它就在⼀个函数作⽤域下⾯。这些变量 只能在函数内部访问，不能在函数以外去访问
  全局作用域：
    任何不在函数中或是⼤括号中声明的变量，都是在全局作⽤域下，全局作⽤域下声明的变量可以在程序 的任意位置访问
  暂时性死区：


作用域链

当在 Javascript 中使⽤⼀个变量的时候，⾸先 Javascript 引擎会尝试在当前作⽤域下去寻找该变 量，如果没找到，再到它的上层作⽤域寻找，以此类推直到找到该变量或是已经到了全局作⽤域

如果在全局作⽤域⾥仍然找不到该变量，它就会在全局范围内隐式声明该变量(⾮严格模式下)或是直接 报错

必包: -> 函数内部可以访问外部函数作用域的函数。

什么是闭包：
  闭包指的是一个函数可以访问另一个函数作用域中的变量，常见的构造方法，是在一个函数内部定义另一个函数，内部函数可以访问外层的变量。
  外层变量不会被垃圾回收机制回收

注意：
  闭包的原理是作用域链，所以闭包访问的上级作用域中的变量是个对象，其值为其运算结束后的最后一个值。
优缺点：
  优点：避免全局变量污染。
  缺点：容易造成内存泄漏。

垃圾回收机制

垃圾回收（Garbage Collection）是指计算机程序自动管理堆内存的一种机制，通过找出不再被使用的对象并释放其占用的内存空间，从而避免内存泄漏和内存溢出等问题。
通过垃圾回收机制，可以有效地避免内存泄漏和内存溢出等问题，提高程序的健壮性和稳定性。

Event Loop

事件循环（Event Loop）是浏览器或 Node.js 中实现异步编程的一种机制。它的主要作用是监听程序中的各种事件，并根据事件类型和优先级将其分配给不同的任务队列，以实现异步执行。

事件循环包括两个重要的部分：
  任务队列和事件循环线程。

任务队列分为宏任务队列和微任务队列。
  宏任务队列存放着诸如 setTimeout、setInterval、Ajax、DOM 事件  等异步任务，
  而微任务队列则存放着 Promise、async/await 等微任务。

事件循环线程是一个独立的线程，负责监听任务队列中是否有任务待执行。当任务队列中存在任务时，事件循环线程会将任务从任务队列中取出并执行。每次执行完一个任务后，事件循环会检查微任务队列中是否还存在未执行的任务，如果有，则先执行所有的微任务，再继续处理宏任务队列中的任务。这样可以保证在当前任务执行完毕之前，所有微任务都已经执行完毕了。

浏览器的异步线程有哪些？

在浏览器中，异步编程常用的机制包括以下几种：
  定时器（Timer）机制：通过 setTimeout 和 setInterval 等方法，在指定的时间后执行回调函数。
  Ajax 异步请求：通过 XMLHttpRequest 或 fetch 等 API 发送异步请求，返回结果后执行回调函数。
  事件监听（Event Listener）机制：通过 addEventListener 等方法注册事件监听器，在特定事件发生时执行回调函数。
  Promise 和 async/await：使用 Promise 可以构建异步链式调用，而 async/await 更是让异步编程更加简单和直观。

总之，浏览器中的异步线程有很多种，每种机制都有其自身的优缺点和适用场景。在实际开发中，需要根据具体的业务需求和性能考虑，选择合适的异步机制和线程，以达到最佳的效果。

js 是单线程语言

js是一门单线程语言。这意味着在一个js程序中，同一时间只能执行一个任务。一旦任务呗开始执行，他必须执行完成才能开始下一个任务。
虽然js是单线程的但是他有异步机制，如(定时器，延时器，promise，async/await)实现非阻塞式的异步调用
这些机制可以让我们等待某些任务执行完毕后，将任务放到任务队列等到js引擎空闲后继续执行

js 模块化（前端模块化，后端模块化）

js模块化是指将复杂的代码按照一定规则封装成单个文件或模块，使得程序结构更加清晰、易于维护和扩展。

模块化分为两种：(前端模块化，后端模块化)
  前端模块化：
    CommonJS：最初是为Node.js设计的模块化规范，使用require(),和module.exports进行模块定义和导出。
    AMD：（Asynchronous Module Definition）异步导出模块，主要用于浏览器环境，使用define()定义模块，require()加载模块。
    ES6：ES6引入了官方的模块化规范，使用import和export关键字进行模块定义和导出。
  后端模块化：
    CommonJS：在Node.js中任然是主流的模块化规范，在每个模块种使用require()和module.export()进行模块定义和导出
    ES6：与前端的ES6模块化相同，使用import和export关键字就行模块定义和导出

数据状态管理器的三大原则

单一数据源（Single source of truth）：所有的数据都保存在一个单一的数据源中。这样做可以使得不同部分的代码共享同一个数据，而不需要在多个组件之间进行无谓的传递，避免了数据冗余和不一致的问题。

状态不可变性（State is read-only）：状态不可变意味着状态只能通过纯函数来修改，而不能直接修改状态。这样可以避免意外地修改状态，从而保证数据的可预测性和一致性。

变化由纯函数完成（Changes are made with pure functions）：所有对状态的修改必须通过调用纯函数来完成。纯函数只接受输入，并输出新的结果，不会修改任何传入的参数，也不会产生副作用。这样可以确保状态的改变是可追溯的，并且便于测试和调试。

同步和异步的区别是什么？异步结果不确定？

同步是指程序按照代码的顺序依次执行，每一行代码必须等到上一行代码执行完成之后才能执行。在同步编程中，如果某个任务需要等待另一个任务完成才能继续执行，那么当前任务就会被阻塞，直到另一个任务完成。

异步则是指程序不按照代码的顺序执行，而是通过回调函数或事件处理函数等方式来处理任务的结果。在异步编程中，当某个任务需要等待另一个任务完成时，程序不会被阻塞，而是继续执行下一行代码，在另一个任务执行完成后再执行回调函数或事件处理函数等，以获取任务的结果。

因为异步任务通常是由操作系统或其他进程来执行，所以异步任务的执行时间可能受多种因素影响，例如网络延迟、计算机负载、硬件故障等，因此异步任务的结果并不总是确定的。我们无法保证异步任务的执行顺序和时间，但可以通过合适的异步编程模式和技术手段来最大化利用异步执行带来的效率优势，并减少或避免异步结果不确定的问题。

总之，同步和异步是两种不同的编程模型，它们之间的区别在于程序执行的顺序和任务处理的方式。异步编程可以提高程序的效率和响应速度，但也带来了异步结果不确定的问题，需要合理地使用异步编程模式和技术手段来解决这些问题。


vue3 中 setup 在什么时候触发

在 Vue 3 中，组件的 setup 函数是在组件实例创建之前执行的。具体来说，在组件被挂载到 DOM 上之前，Vue 会先执行 setup 函数，将 setup 函数返回的数据作为响应式对象注入到组件实例中，然后再开始执行组件的生命周期钩子函数。

setup 函数的执行时机和 Vue 2 中的 beforeCreate 生命周期钩子类似，但有一些区别：

setup 函数的参数包括了组件的 props、attrs、context 等属性，并且不能访问组件实例（即 this）；
setup 函数中可以使用 Vue 3 提供的新 API，如 ref、reactive、computed 等，用于创建响应式数据；
setup 函数需要返回一个对象，该对象中的属性和方法将被注入到组件实例中，可以在组件中通过 this.xxx 的方式进行访问。
总之，在 Vue 3 中，setup 函数是一个非常重要的概念，它负责创建并初始化组件的状态，并将其注入到组件实例中。由于 setup 函数的执行时机很早，因此可以在其中进行一些预处理或者初始化操作，从而减少在组件生命周期钩子中的计算量，提高性能和效率。
