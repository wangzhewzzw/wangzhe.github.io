v-show 和 v-if 的区别

两者都是判断类型都是布尔值
v-if
如果为真显示
如果为假删除，不会在页面渲染
v-show
如果为真显示
如果为假用display：none控制隐藏但还是会在页面渲染
总结:切换的频繁的话用v-show，如果不频繁就一次性的话用v-if

使用场景

根据条件切换显示隐藏或销毁dom元素

为何 v-for 中要用 key

在v-for指令中使用key能够帮助Vue跟踪列表中每个节点的身份，并且尽可能地复用已有的元素而不是重新渲染。如果没有使用key，那么当列表中的顺序、数量或者内容发生变化时，Vue可能需要销毁和重建大部分的DOM元素来更新视图，这样会导致页面性能下降且用户体验不佳。

描述 Vue 组件生命周期(有父子组件的情况)

Vue组件的生命周期可以分为以下几个阶段：

创建阶段（Creation）

beforeCreate：在实例创建之前调用，该钩子函数无法访问到实例中的数据和方法。
created：在实例创建完成后调用，可以访问到实例中的数据和方法，但是不能保证所有子组件都已经创建完成。
beforeMount：在挂载元素之前调用，此时模板编译已完成，但尚未将模板渲染到页面上。

更新阶段（Update）

beforeUpdate：在更新之前调用，此时虚拟DOM已经完成比较，即将开始重新渲染。
updated：在更新完成之后调用，此时组件已经重新渲染完毕。
销毁阶段（Destruction）

beforeDestroy：在实例销毁之前调用，此时实例仍然完全可用。
destroyed：在实例销毁之后调用，此时实例中的所有数据和方法都已经被删除，无法再访问。
在父子组件中，还有一些额外的钩子函数：

父组件创建子组件

beforeCreate
created
beforeMount
mounted

子组件更新

beforeUpdate
updated

父组件更新

beforeUpdate
updated

父组件销毁子组件

beforeDestroy
destroyed
注意，当父组件被销毁时，它的所有子组件也会被递归地销毁。因此，在处理父子组件的生命周期时，需要特别注意它们之间的关系。

Vue 组件如何通讯

父子组件通讯

Props/emit：父组件通过props向子组件传递数据，子组件通过$emit触发事件，将数据传递回父组件。
Refs：父组件通过ref获取到子组件的引用，从而直接调用子组件的方法或访问子组件的数据。

兄弟组件通讯

EventBus：使用Vue实例作为事件总线，在一个组件中触发事件，另一个组件监听该事件并执行相应操作。
Vuex：状态管理库，包含了全局状态、状态变更、状态派生等功能，使得多个组件共享同一个状态。

跨级组件通讯

Provide/Inject：祖先组件通过provide提供数据，后代组件通过inject注入数据，从而实现跨级组件通讯。

attrs/listeners：父组件通过v-bind="$attrs"和v-on="$listeners"将所有属性和事件绑定到子组件上，子组件可以从$attrs中访问这些属性，从$listeners中监听这些事件。

描述组件渲染和更新的过程

组件渲染和更新是Vue的核心功能之一，其过程可以分为以下几个步骤：

编译阶段

在编译阶段，Vue会对组件的模板进行编译，生成对应的render函数或者虚拟DOM树。如果是使用template标签来定义模板，则需要将模板转换为render函数；如果是使用jsx语法，则不需要经过编译。

创建组件实例

在创建组件实例时，Vue会调用组件类的构造函数，并传入初始化参数。在构造函数中，Vue会对组件实例进行初始化，包括数据初始化、事件绑定、生命周期钩子等操作。

执行beforeCreate和created钩子

在组件实例创建完成后，Vue会依次执行beforeCreate和created钩子。在这两个钩子中，我们可以访问到组件实例中的所有选项，但是此时还没有访问到$el属性，因为组件还没有被挂载到DOM树上。

挂载组件

Vue会自动将组件实例的render函数或者虚拟DOM树渲染成真实的DOM节点，并将该节点添加到父组件中。在这个过程中，Vue会依次执行beforeMount和mounted钩子。在beforeMount钩子中，我们可以访问到组件的真实DOM节点，但是此时节点还没有被插入到页面中；在mounted钩子中，节点已经被插入到页面中，我们可以通过$el属性访问到节点。

更新组件

当组件中的数据发生变化时，Vue会自动重新渲染组件，并将变化反映到视图中。在更新过程中，Vue会依次执行beforeUpdate和updated钩子。

销毁组件

当组件需要被销毁时，Vue会依次执行beforeDestroy和destroyed钩子。在这两个钩子中，我们可以进行一些清理工作，比如取消事件监听、清除定时器等操作。

通过以上几个步骤，Vue实现了组件的渲染和更新。需要注意的是，在组件更新时，Vue会使用虚拟DOM技术进行高效的比对和更新，从而提高页面性能和响应速度。

双向数据绑定 v-model 的实现原理

其实现原理主要依赖于以下两个方面：

v-model语法糖

在模板中使用v-model时，Vue会将该指令转换为一对带有v-bind和v-on指令的表达式。例如，在一个输入框中使用v-model时，实际上相当于以下代码：

html
<input v-bind:value="message" v-on:input="message = $event.target.value">
这里的v-bind用于将组件的value属性绑定到message变量上，v-on则用于监听输入事件并将输入框的值赋给message变量。因此，通过这种方式，Vue实现了将用户输入的内容更新到组件实例中的目的。

组件实例中的$emit方法

在监听用户输入时，Vue会通过组件实例上的$emit方法触发一个名为input的自定义事件，并将用户输入的值作为参数传递回父组件。在父组件中，我们可以监听该事件，并将获取到的值更新到组件实例中，从而实现将组件实例中的数据反映到视图中的目的。

例如，在一个自定义文本框组件中，我们可以定义如下的$emit方法：

javascript
this.$emit('input', event.target.value)
然后在父组件中监听该事件，并更新组件实例中的数据：

html
<my-textbox v-model="message"></my-textbox>

<script>
export default {
  data () {
    return {
      message: ''
    }
  }
}
</script>
在这个例子中，我们监听了my-textbox组件的input事件，并将获取到的值赋给message变量，从而实现了双向数据绑定。

综上所述，v-model指令主要依赖于语法糖和自定义事件实现双向数据绑定，通过这种方式，Vue让开发者更加方便地实现表单控件与数据模型之间的同步。
